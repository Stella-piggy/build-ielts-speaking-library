# 雅思口语素材库

`build-ielts-speaking-library` 是一个面向 IELTS Speaking 备考者的个人 Codex Skill。它不会批量制造通用模板答案，而是从你的真实经历、偏好和观点出发，建立一份能够持续更新、反复串题的个人口语素材库。

它尤其适合这样的学习者：题库很多但不知道如何整理；准备了不少答案却记不住；同一段经历不知道能用于哪些题；想把表达提升到 6.5—7 分目标难度，又不希望答案变得书面、生硬或不像自己。

> 分数区间只是表达难度参考，不代表或承诺真实考试成绩。

## 它解决什么问题

| 常见痛点 | Skill 的处理方式 |
| --- | --- |
| 题库重复、相似题很多，逐题准备成本太高 | 解析 Part 1、Part 2、Part 3，保留原题，同时识别重复题、相似题和可复用主题 |
| 模板答案听起来不像本人，也很难记住 | 先采访真实经历，只把用户确认过的事实写入素材库 |
| 为了串题，容易把不合适的故事硬套到题目上 | 每个映射标记为“自然覆盖”“调整后可用”或“不建议使用”，并说明强行串题风险 |
| 所谓高分版只是机械替换高级词 | 两个版本在逻辑展开、衔接、改述、搭配和句式上形成真实差异 |
| 纠错后内容完全变成老师或 AI 的表达 | 先给最小修改版，再给提升版，并保留用户原意和已经能自然说出的结构 |
| 每次练习都从零开始，素材无法积累 | 持续维护可复制的 `ielts-speaking-library.md`，记录素材卡、题目映射和个人语言风格 |

## 核心功能

- 解析粘贴或上传的 IELTS Speaking 题库，识别 Part、原题、回答要求和所需素材类型。
- 合并严格重复题，区分“主题相似但回答任务不同”的题目，避免错误去重。
- 根据题库覆盖范围规划 6—8 个高信息密度采访模块，每个模块只问一个主问题，必要时最多追问两次。
- 在写入前展示候选事实，只有经过用户确认的内容才会进入素材库。
- 分别建立三类资产：
  - Part 1：个人画像卡，例如偏好、原因、例子和变化。
  - Part 2：个人故事卡，例如背景、困难、行动、结果、情绪和反思。
  - Part 3：观点卡，例如立场、原因、现实例子、让步和结论。
- 建立素材卡与每一道原始题目的映射，并报告覆盖情况、缺失素材和串题风险。
- 用户选定具体题目后，按需生成：
  - 5.5—6 分稳定表达版；
  - 6.5—7 分灵活自然版；
  - 中文逻辑、英文关键词、版本差异和 3—5 个值得积累的表达。
- 对用户自己的文字或口语练习进行纠错，先检查是否答题，再处理逻辑、语法、搭配、重复用词和翻译腔。
- 根据真实练习表现逐步调整句长、词汇难度和表达风格。

## 设计原则

- 不虚构用户经历；信息不足时追问或明确标记缺失。
- 不靠生僻词、俚语或复杂句堆砌制造“高分感”。
- 不机械逐词替换同义词，优先保证整句话自然、准确、容易复述。
- Part 3 必须有观点、理由和例子，不能只用个人故事代替论证。
- 没有合适素材时补充采访，不为了提高覆盖率强行串题。
- 文字回答不评价发音；只有环境能够处理真实音频时，才提供参考性的清晰度、重音、节奏、语调和连读反馈。

## 安装

在 Codex 中直接发送：

```text
请使用 $skill-installer 安装这个 Skill：
https://github.com/Stella-piggy/build-ielts-speaking-library
```

安装完成后，从下一轮对话开始即可使用。也可以手动将本仓库完整复制到 Codex 的个人 Skills 目录，并确保目录名为：

```text
build-ielts-speaking-library
```

## 快速开始

### 1. 第一次建立素材库

粘贴或上传你的题库，然后发送：

```text
使用 $build-ielts-speaking-library 分析这份雅思口语题库。
先展示题库解析、主题聚类和采访计划，再一次采访一个主题。
不得虚构我的经历，所有事实写入前都要让我确认。
```

你可以补充以下信息帮助校准：

```text
目标难度：6.5—7 分
访谈语言：中文或中英混合
答案语言：英文
希望风格：自然、真诚、容易复述
当前身份和主要经历：……
兴趣和常见生活场景：……
```

题库可以直接粘贴，也可以上传当前 Codex 环境能够读取的文档或表格。Skill 会先分析和采访，不会一次性生成整本题库的答案。

### 2. 使用素材库练习某道题

上传或保留上一次生成的 `ielts-speaking-library.md`，然后发送：

```text
使用 $build-ielts-speaking-library，基于我的素材库练习这道 Part 2 题：
[粘贴原题]

请生成两个难度版本，并告诉我最需要注意的一个问题。
```

### 3. 提交自己的英文回答进行纠错

```text
这是我刚才实际说出的回答：
[粘贴英文回答或上传音频]

请先判断我是否回答了题目，再给最小修改版和 6.5—7 分提升版。
只提炼 3—5 个最值得记忆的表达，并根据我的表现调整后续难度。
```

如果只有文字或自动转写文本，Skill 不会假装评价发音。

### 4. 更新已有故事

```text
请更新素材库中的故事卡 STORY-001。
需要补充或修正的真实信息是：……

先列出准备修改的事实让我确认，再更新题目映射和完整素材库。
```

你也可以不用卡片编号，直接说故事标题或描述发生了什么。Skill 会保留稳定 ID，并在事实冲突时先与你核对。

## 一次完整使用会得到什么

```text
题库
  → 原题解析与去重审计
  → 主题聚类
  → 6—8 个采访模块
  → 用户确认事实
  → 画像卡 / 故事卡 / 观点卡
  → 题目与素材映射
  → 覆盖报告与缺失素材
  → 选题后生成双版本答案
  → 练习纠错并更新个人风格
```

最终的 `ielts-speaking-library.md` 是可持续使用的主文件。下次练习时重新上传它，就可以继续使用已有故事、更新观点或补充新的题库。

## 输出示例

针对一道人物或经历类题目，Skill 不会直接杜撰完整答案。它会先确认人物、背景、困难、行动、结果、真实感受等信息，再生成：

1. 中文回答逻辑；
2. 英文关键词提纲；
3. 5.5—6 分稳定表达版；
4. 6.5—7 分灵活自然版；
5. 两个版本在逻辑、衔接、词汇和句式上的差异；
6. 3—5 个自然表达及适用语境；
7. 可迁移到其他题目的表达；
8. 本次练习最需要注意的问题。

更完整的合成案例见 [`references/examples.md`](references/examples.md)。

## 文件结构

```text
build-ielts-speaking-library/
├── SKILL.md
├── agents/
│   └── openai.yaml
└── references/
    ├── answer-rubric.md
    ├── asset-schemas.md
    ├── examples.md
    └── interview-and-mapping-rules.md
```

- `SKILL.md`：核心运行流程与资源调用规则。
- `answer-rubric.md`：双版本答案、评分维度和练习纠错规则。
- `asset-schemas.md`：画像卡、故事卡、观点卡和主素材库格式。
- `interview-and-mapping-rules.md`：题库解析、采访、确认和串题映射规则。
- `examples.md`：重复题、短回答、事实冲突、无合适素材和答案分层等测试案例。

## 使用边界与隐私

这个仓库只包含 Skill 规则，不包含任何真实学习者的题库、音频、个人资料或素材库。你在使用过程中生成的 `ielts-speaking-library.md` 可能包含个人经历，请自行决定保存和分享范围。

本 Skill 是个人学习辅助工具，不提供官方 IELTS 评分，也不替代真实口语交流、教师反馈或正式模考。
